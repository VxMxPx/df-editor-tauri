# Requirements

One file is one coherent feature or change.

```md
# Feature name

Short description of the outcome.

## Requirements

- Observable behavior.

## Out of scope

- Explicitly excluded work.

## Acceptance

- Steps that prove the change works.
```

Ask to implement a file by name, for example: `Implement requirements/001-explorer-tabs.md`.

Before implementation, review the requirement, resolve unclear decisions, and update the file for clarity. When the work is complete, offer to create logical commits.
