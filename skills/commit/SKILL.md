# Commit

1. Inspect `git status` and diffs. Group only related changes into small, logical commits; do not include unrelated work.
2. If a completed requirement is included, move it from `requirements/` to `requirements/done/` in that same commit.
3. Ask before committing when the intended grouping, ownership, or requirement completion is unclear.
